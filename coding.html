Book Review - "The Art of Clean Code" by Christian Mayer

June 6, 2022

I read this book as part of Project Code, on recommendation from the Python foundation. It discusses how to think about software and efficient design, making it a prime book to start with.

Though the book's examples were written in Python, it's driving philosophy is useful in any language. For example, the SOLOMON Project is a massive exercise. To keep myself from getting overwhelmed, I must think about developing only what I need for now.

Even in SOLOMON's early stages, I am thinking about the Unix principles that Mayer summarizes:

  1. Make each function do one thing well
  2. Simple is better than complex
  3. Small is beautiful
  4. Build a prototype as soon as possible
  5. Choose portability over efficiency
  6. Store data in flat text files
  7. Use software leverage to your advantage
  8. Avoid captive user interfaces
  9. Make every program a filter
  10. Worse is better
  11. Clean code is better than clever code
  12. Design programs to be connected with other programs
  13. Make your code robust
  14. Write programs to write programs

Currently I am still reading "HTML & CSS - design and build websites" by Jon Duckett. I have also started "Think Python" by Allen B. Downey.
